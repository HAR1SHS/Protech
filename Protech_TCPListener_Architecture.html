pt-tcp-listener — Cloud-Native Ingestion

The first hop of DecodingEngine: a thin, multi-tenant Python 3.12 asyncio service that listens on plant-floor TCP ports, validates the legacy meter checksum, and publishes the raw frames to Azure Event Hubs. It is the modern replacement for the on-prem .NET PortApps gateways — all decoding now happens downstream in Databricks.

Plant meters  →  TCP listener (this service)  →  Event Hubs  →  Databricks  →  Delta + Lakebase

1Where It Fits — vs. Legacy PortApps

PortApps did everything on-premise (accept → checksum → decode → SQL stored procs). pt-tcp-listener takes only the first hop and pushes meaning downstream.

2What It Does (and Deliberately Does Not)

✓ Does

  • Binds N TCP servers (one per port) in a single asyncio loop
  • Per-IP concurrency cap, reads to the * terminator
  • Validates the 5-digit checksum, computes msg_key = sha1(raw)
  • Publishes raw bytes to the matching Event Hubs topic with headers
  • Replies with an ACK (005A) on success
  • HTTP :8080 with /health · /ready · /metrics
  • Graceful SIGTERM/SIGINT drain + Kafka flush

✗ Does NOT

  • Parse / decode the payload (no catalog lookup, no float conversion)
  • Touch SQL Server
  • Send an explicit NACK on failure — failures are silent so the meter times out and retransmits (preserves at-least-once)

4Packet Envelope & Checksum

Same legacy $…* frames as PortApps. This service only validates the envelope + checksum — it never looks at the payload.

$ <body bytes> <5-digit checksum> *
 ↳ '$' excluded from sum  ·  checksum = sum(body bytes) % 100000  ·  '*' = terminator
Validity: packet starts with $ and ends with *; the 5 decimal digits immediately before * are the declared checksum; valid when sum(bytes between '$' and the checksum) % 100000 == declared. A simple additive checksum, intentionally byte-for-byte compatible with the legacy .NET/VB code so meters need no change. Canonical regression test uses a real McDonald's packet: $00612100DMcDonalds1…06319*.

7Reliability & Delivery Semantics

At-least-once

Event Hubs is treated as a replayable log; no exactly-once at the producer.

Idempotent dedup

msg_key = sha1(raw) lets Delta MERGE / Postgres ON CONFLICT dedup retransmits & retries.

Silent-failure contract

Oversize / checksum / publish failures return no ACK → meter retransmits (legacy behaviour, preserved).

Graceful shutdown

SIGTERM → stop accepting → drain handlers → flush producers → exit (~30 s on Container Apps).

Outage tolerance

~1M-message in-producer buffer (≈13 min at 1,250 msg/s) rides out brief EH degradation.

Stateless scale-out

No local persistence; all state in Event Hubs. Multiple replicas behind the same ports.
