A complete walkthrough of the ProTech Admin Console: the persistent shell every screen lives in, the three-role model that trims what you see, then each module top to bottom under its sidebar group — MAIN, OPERATIONS and ADMIN — with its route, screens, wizards and the create / read / update / archive actions it exposes. This is the panel's information architecture, not the one-time onboarding sequence.

Platform Admin / Super User

Sees: everything, fleet-wide. The operator "root" — every org, store, device, catalog & access rule. Only role that can edit identity codes (Org Code, Store Code, Communication ID).

Admin

Sees: everything, platform-wide. Mirrors Platform Admin across all tenants — but cannot edit identity codes ("Only a Platform Admin can change this").

Organization Admin

Sees: your organization only. Own org's stores, users & store-level config. No Master Data / Catalogs — the menu quietly trims to Store + Preferences.
